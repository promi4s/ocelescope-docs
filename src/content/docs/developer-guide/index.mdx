This section should be for people modifying the Ocelescope platform, not writing plugins.

The story should explain how the repository is organized, how backend and frontend services interact, how API clients are generated, how documentation is built, and how to run the development stack locally.

Keep this separate from Plugin Development so plugin authors are not forced through platform internals.
