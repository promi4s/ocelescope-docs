This section should be the main path for researchers and developers who want to bring their own analysis code into Ocelescope.

The story should be: start from a template, write Python logic, declare plugin metadata, expose methods with type hints, define inputs and resources, add visualizations, package the plugin as a ZIP, upload it, and run it in the UI.

The overview page should make the workflow feel short and concrete before sending readers into details.
