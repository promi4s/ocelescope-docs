This page should explain runtime plugin loading from the user perspective.

The story should be: obtain a plugin ZIP, upload it through the plugin page, confirm that Ocelescope recognized the plugin metadata and methods, then run one method against an available OCEL or resource.

Mention that plugin methods describe their inputs declaratively, which lets the frontend generate the correct form automatically.
