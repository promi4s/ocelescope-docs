## From log to insight

Ocelescope gives users a focused workflow for working with object-centric event data.

<Steps>
1. **Start Ocelescope.** 

    Run frontend and backend with a reproducible container-based setup.
2. **Load an OCEL.** 

    Import object-centric event logs and manage them inside one session.
3. **Inspect the log.** 

    Explore events, objects, relations, and summary information before analysis.
4. **Filter and prepare.** 
 
    Focus on the relevant object types, event types, relations, and time ranges.
5. **Run plugins.**
 
    Apply OCPM methods through generated forms based on plugin declarations.
6. **Export results.** 

    Continue with generated OCELs, resources, visualizations, and output files.
</Steps>

## Plugins and modules

Ocelescope supports two extension paths.

<CardGrid>
	<Card title="Plugins" icon="add-document">
		Best for lightweight OCPM functionality: algorithms, transformations, resources, visualizations, and reusable analysis functions that can be zipped and loaded at runtime.
	</Card>
	<Card title="Modules" icon="setting">
		Best for larger tools with custom frontend views, backend endpoints, and specialized workflows that require actual code adjustments in the application.
	</Card>
</CardGrid>

Use plugins when you want to add a method quickly.
Use modules when Ocelescope should become the foundation for a larger OCPM application.
